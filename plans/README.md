## Findings considered and rejected

- **Runtime-to-services decomposition**: rejected by the current R2 ADR. The
  runtime is a coordinator facade over already-separated engines; no second
  consumer or dependency-injection driver exists.
- **Per-mode Runtime split**: rejected by ADR. One revision clock and
  cross-strategy edges are intentional differentiators, not accidental coupling.
- **Optimize `AccumulatorCommitHook::after_abort`'s quadratic scan now**:
  rejected for current workloads. The repository's measured evidence says real
  consumers use roughly 1–2 accumulators per memo; reopen only around 50+
  accumulators with frequent aborts.
- **Treat `Derived::map_no_backdate`'s permissive read as an observer-allocation
  performance bug**: rejected after source verification. Current strict and
  permissive reads share `read_honest()` internally; the difference is the
  tracked-context guard, not an extra observer lifecycle.
- **Expose the private static/applicative Derived fast path now**: rejected by
  the accepted public-surface ADR until `Expr`, an attachment pipeline, or a
  downstream UI supplies a measured driver. The shipped dynamic `map` helpers
  do not reopen that decision.
- **Consolidate CI jobs solely because installation is duplicated**: not
  commissioned without timing evidence. R15 preserves the measurement question;
  plan 006 hardens provenance without changing job isolation.
